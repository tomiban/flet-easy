    JWT Básico¶

    JWT es un estándar para transmitir datos de forma segura entre partes en formato JSON. Se compone de tres partes: el encabezado, el payload y la firma. Se usa comúnmente para autenticación y autorización en aplicaciones web y servicios API. Los JWT son compactos, autónomos y fáciles de usar en entornos distribuidos.

    Flet-Easy contiene una integración básica para usar JWT de manera simple y rápida, al integrar JWT en la aplicación puedes configurar el tiempo de sesión del usuario, así como una expiración automática de sesión. Para agregar JWT necesitaremos las siguientes configuraciones.

    En la clase FletEasy debemos configurar los siguientes parámetros:

    • El valor de auto_logout es falso por defecto (cierra sesión automáticamente).
    • Para configurar la secret_key es necesario usar la clase SecretKey de FletEasy.

    Configuración¶

    Algoritmo HS256¶

    main.py
    import flet_easy  as fs

app = fs.FletEasy(
    route_init="/login",
    route_login="/login",
    secret_key=fs.SecretKey(
        algorithm=fs.Algorithm.HS256,
        secret=SECRET_KEY
        ),
    auto_logout=True,  # Activa el cierre automático de la sesión
)

    Algoritmo RS256¶

    Info

    Para usar el algoritmo RS256 debes tener instalada la librería cryptographic.

    main.py
    import flet_easy  as fs

app = fs.FletEasy(
    route_init="/login",
    route_login="/login",
    secret_key=fs.SecretKey(
       algorithm=fs.Algorithm.RS256,
       pem_key=fs.PemKey(private=PRIVATE_KEY, public=PUBLIC_KEY)
    ),
    auto_logout=True,  # Activa el cierre automático de la sesión
)

    Obtener secret_key¶

    FletEasy proporciona una clase llamada EasyKey para obtener fácilmente una secret_key para usar, luego podemos copiarla a un archivo o usarla como variables de entorno.

    import flet_easy as fs

key = fs.EasyKey()

# --- HS256
SECRET_KEY = key.secret_key()

# --- RS256
PRIVATE_KEY = key.private_key()
PUBLIC_KEY = key.public_key()

    Cómo usarlo¶

    Después de haber configurado la secret_key, podemos comenzar a configurar el uso de JWT. Para esto vamos a requerir el uso del método login de Datasy (data), se usará como normalmente lo usaríamos sin usar JWT, pero usaremos el parámetro time_expiry y value que tendrá que ser un diccionario obligatoriamente.

    En este ejemplo vamos a hacer muy similar con el ejemplo de Route-protection, solo hemos configurado la secret_key, usado el parámetro time_expiry del método login y usado la función decode de FletEasy para obtener el payload almacenado en el almacenamiento del cliente decodificado.

    main.py
    from datetime import timedelta

import flet as ft
import flet_easy as fs
from sensitive import SECRET_KEY

db = []

app = fs.FletEasy(
    route_init="/login",
    route_login="/login",
    secret_key=fs.SecretKey(
        algorithm=fs.Algorithm.HS256,
        secret=SECRET_KEY
        ),
    auto_logout=True,  # Activa el cierre automático de la sesión
)

@app.login
def login_x(data: fs.Datasy):
    # decodificar payload
    value = fs.decode(key_login="login", data=data)

    print("value:", value)

    """ Verificamos si el nombre de usuario que está almacenado en el navegador
    está en la base de datos simulada. """
    if not value:
        return False
    elif value.get("user") in db:
        return True


@app.page(route="/dashboard", title="Dashboard", protected_route=True)
def dashboard_page(data: fs.Datasy):
    return ft.View(
        controls=[
            ft.Text("Dash", size=30),
            # Eliminamos la clave que hemos registrado previamente
            ft.ElevatedButton(
                "Logaut",
                on_click=data.logout("login")
                ),
            ft.ElevatedButton(
                "Home",
                on_click=data.go("/login")
                ),
        ],
        vertical_alignment="center",
        horizontal_alignment="center",
    )


@app.page(route="/login", title="Login")
def login_page(data: fs.Datasy):
    # crear usuario almacenado de inicio de sesión
    username = ft.TextField(label="Username", width=200)

    def store_login(e):
        db.append(username.value)  # Agregamos a la base de datos simulada

        """Primero los valores deben almacenarse en el navegador, luego en el
        decorador login se debe recuperar el valor a través de la clave usada
        y luego se deben usar validaciones."""
        data.login(
            key="login",
            value={"user": username.value},
            next_route="/dashboard",
            time_expiry=timedelta(seconds=10),
        )

    return ft.View(
        controls=[
            ft.Text("login", size=30),
            username,
            ft.ElevatedButton(
                "store login in browser",
                on_click=store_login
                ),
            ft.ElevatedButton(
                "go Dasboard",
                on_click=data.go("/dashboard")
                ),
        ],
        vertical_alignment="center",
        horizontal_alignment="center",
    )

app.run()

    decode¶

    Decodifica el jwt y actualiza las sesiones del navegador.

    Parámetros a usar:

    • key_login: clave usada para almacenar los datos en el cliente, también usada en el método login de Datasy.
    • data: Instancia de objeto de la clase Datasy.

    Info

    _ Soporta async, ejemplo: decode_async. _ Si la función a usar es async se recomienda usar decode_async para evitar errores.

    Nota

    Las funciones decode y decode_async se pueden usar en otras partes del código, por ejemplo: Middleware
